# Migration Plan: Add Tavily as parallel search provider to Rust WebSearchTool

**Unit ID:** rust-web-search-brave-to-tavily
**Search Provider:** Brave Search API
**Migration Strategy:** additive

## Objective

Extend the existing WebSearchTool in src/tools/web_search.rs to support Tavily as a configurable/fallback search backend alongside the existing Brave Search integration, controlled via a new Cargo feature flag and runtime configuration key.

## Implementation Steps

1. Add TAVILY_SEARCH_ENDPOINT constant and Tavily request/response structs to web_search.rs
2. Extend resolve_api_key to also read TAVILY_API_KEY env var and ctx.extra["tavily_api_key"]
3. Add provider-selection logic: if Tavily key is present (and optionally SEARCH_PROVIDER=tavily), route to Tavily HTTP path; otherwise fall back to existing Brave path
4. Map Tavily response fields (results[].title, url, content) to the shared SearchResult struct
5. Add a web_search_tavily Cargo feature (or extend web_search) so the Tavily code path can be compiled in opt-in
6. Add unit tests mirroring existing Brave tests for the Tavily error path (missing key, invalid input)

## Files to Modify

### `src/tools/web_search.rs`
- Add Tavily API client structs, endpoint constant, and provider-selection logic (env var or ctx key) that falls back to Brave when Tavily is not configured.

### `Cargo.toml`
- Add optional tavily feature flag (e.g. web_search_tavily) and any new dependencies if a dedicated Tavily crate is used; otherwise reqwest already covers the HTTP call.

## Dependency Changes

- No new crate required — reqwest (already optional dep for web_search feature) handles Tavily HTTP calls

## Environment Variable Changes

- Add TAVILY_API_KEY — Tavily API key; if set, enables Tavily provider path
- Add SEARCH_PROVIDER (optional) — set to 'tavily' or 'brave' to force a specific provider when both keys are present; defaults to Tavily when TAVILY_API_KEY is set

## Notes

Existing BRAVE_API_KEY env var and ctx.extra["brave_api_key"] must be kept unchanged. The Tavily Search API endpoint is https://api.tavily.com/search (POST, JSON body with api_key, query, max_results fields). Tavily returns results as an array under the 'results' key with title, url, content fields.
